Museum Data Integration and Analysis Using Excel, Python and SQL Server

  • CSV files containing data about famous paintings, artists, and museums are uploaded to SQL server using SQL Alchemy in Python. Queries are then performed in SQL Server to analyse museum information.
  • The data is initially explored in Excel, where it is found that the opening and closing times in the museum_hours CSV file is formatted incorrectly. Power Query is utilised to convert these columns into appropriate time formats so they can be recognised by SQL. Using Power Query means that the formatting can be applied automatically to any new data introduced.
  • In Python, a connection to an SQL Server database is established using SQLAlchemy. The script iterates over a dictionary of the eight CSV files and their corresponding table names, reading each file into a Pandas DataFrame and uploading the data into the specified SQL tables. This Python script demonstrates the ease of moving data from CSV files into an SQL database.
  • In SQL Server, queries are performed to find museum details, museums open on certain days, and the museum with the longest opening hours.
  • Subqueries and stored procedures are utilised to improve the efficiency of the queries.

  • LSTM model constructed and trained with manual hyperparameter tuning for optimal results.
  • XGBoost model is built with grid search and cross-validation for optimal hyperparameters, providing feature importance to overcome ML black-box problem.

Excel Coffee Sales Dashboard

  • In this project I used an Excel workbook with three sheets containing data about customers, orders and products of a coffee company to create an interactive dashboard of coffee sales.
